# Dashboard 数据可视化升级设计方案

## 概述

对 NetWork AI 平台的 Dashboard 页面进行全面升级，添加数据可视化模块，提升用户数据洞察能力。

## 设计目标

- 保持赛博朋克主题风格
- 添加 4 个核心数据可视化模块
- 使用 Semi Design Charts 组件库保持 UI 一致性
- 响应式布局，适配不同屏幕尺寸

## 数据模块设计

### 1. 用量趋势折线图 (Usage Trend Line Chart)

**位置**：左侧 2/3 宽度
**功能**：
- 显示 7 天 Token 消耗趋势
- X 轴：日期
- Y 轴：Token 消耗量
- 悬停显示详细数据
- 支持时间范围切换（7天/30天）

**配色**：
- 主线条：`#00d4ff` (primary)
- 填充：`rgba(0, 212, 255, 0.1)`
- 网格线：`#1e2732`

### 2. 模型分布饼图 (Model Distribution Doughnut)

**位置**：右侧 1/3 宽度
**功能**：
- 显示各模型使用占比
- 支持自定义颜色映射
- 悬停显示百分比和具体数值
- 图例显示模型名称

**颜色映射**：
| 模型 | 颜色 |
|------|------|
| GPT-4o | `#10B981` |
| Claude | `#8B5CF6` |
| Gemini | `#F59E0B` |
| DeepSeek | `#3B82F6` |

### 3. 请求量统计面板 (Request Stats Panel)

**位置**：左下角
**功能**：
- 今日请求数
- 本月累计请求数
- 成功率指示器（带进度环）
- 失败率显示

**成功率指示器**：
- 绿色（>95%）：`#30D158`
- 黄色（90-95%）：`#FFD60A`
- 红色（<90%）：`#FF453A`

### 4. 余额预警面板 (Balance Alert Panel)

**位置**：右下角
**功能**：
- 余额状态指示（充足/即将用尽/不足）
- 当前余额显示
- 额度消耗进度条
- 充值引导按钮（当余额不足时显示）

**阈值**：
- 充足：余额 > 20%
- 即将用尽：余额 5-20%
- 不足：余额 < 5%

## 技术实现

### 依赖

```json
{
  "@douyinfe/semi-ui": "^2.69.1",
  "@douyinfe/semi-icons": "^2.63.1",
  "@douyinfe/semi-illustrations": "^2.65.0"
}
```

### 组件结构

```
src/pages/Dashboard/
├── index.jsx          # 主页面
├── Dashboard.css      # 样式
├── components/
│   ├── UsageTrendChart.jsx    # 用量趋势图
│   ├── ModelDistribution.jsx   # 模型分布饼图
│   ├── RequestStats.jsx        # 请求量统计
│   └── BalanceAlert.jsx        # 余额预警
└── hooks/
    └── useDashboardData.js     # 数据获取 hook
```

### API 集成

```javascript
// 获取用量数据
GET /api/usage?start=YYYY-MM-DD&end=YYYY-MM-DD

// 获取用户信息（含余额）
GET /api/user/self

// 获取请求统计
GET /api/usage/stats
```

## 状态管理

使用 React useState + useEffect 本地状态管理，配合 Suspense 处理加载状态。

### 加载状态
- 骨架屏占位
- 动画过渡效果

### 错误状态
- 展示错误提示
- 提供重试按钮

## 动画效果

- 图表加载：渐入动画（300ms ease-out）
- 数据更新：平滑过渡动画
- 悬停交互：轻微放大效果（scale 1.02）
- 进度条：脉冲动画（当余额不足时）

## 响应式断点

| 断点 | 布局 |
|------|------|
| >= 1200px | 4 列指标卡，2 列图表 |
| 768-1199px | 2 列指标卡，2 列图表 |
| < 768px | 1 列堆叠 |

## 实施步骤

1. 安装依赖 `@douyinfe/semi-illustrations`
2. 创建 Dashboard 子组件目录
3. 实现 `useDashboardData` hook
4. 依次实现四个可视化组件
5. 更新主页面布局
6. 添加加载和错误状态
7. 响应式适配测试
